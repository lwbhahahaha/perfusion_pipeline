# perfusion_pipeline

End-to-end CT myocardial perfusion simulation: real patient AIF → BasisSim
forward-projected CT volumes → Mullani–Gould single-volume perfusion map →
interactive 3D viewer.

Built around the [VascularTreeSim.jl](../VascularTreeSim.jl) tree growth +
[FlowContrastSim.jl](../FlowContrastSim.jl) hemodynamics. The pipeline is
**physics-only — no calibration knob**. Inputs are an XCAT digital phantom, a
patient's bolus-tracking AIF (HU vs time), the grown coronary trees, and a
literature-derived capillary-bed resistance. Output is a 3D perfusion map
that reproduces the clinical Mullani–Gould workflow on virtual CT scans.

---

## What is in the pipeline

There are two stacked pipelines that share intermediates. Both consume the
same grown tree CSVs but differ in how they form the V1/V2 images.

### Pipeline A — synthetic gamma AIF, voxel-direct (legacy, kept for reference)
`step1_prep_phantom.jl` → `step2_build_territory.jl` → `step3_voxelize_frames.jl` → `step4_perfusion.py`

* Synthetic clinical gamma-variate as the input function (no patient data)
* Voxelizes Int16 HU frames directly (no BasisSim forward+recon)
* `ARRIVAL_OVERRIDE_S = 1.5` — uniform myo arrival (models healthy
  autoregulation); kept for the legacy run

### Pipeline B — **real patient AIF + BasisSim virtual CT (current canonical)**
`step0_prepare_aif.py` → `step2_build_territory.jl` → `make_basissim_phantoms.jl` → `run_cta_sim_param.jl` (in `phantom_ct_input/run_basis_sim/`) → `step5_perfusion_from_recon.py`

* Loads the patient's `BOLUS_TRACKING_*/AIF_autoGenerated.mat`, baseline-
  subtracts, converts HU → mg/mL iodine
* Uses **per-voxel myocardial arrival** from the fixed-physics FlowContrastSim
  hemo (no override); at-rest tree for REST, max-dilated tree for STRESS
* Forward-projects two cross-product UInt16 phantoms (V1 baseline + V2
  peak) through BasisSimulator and reconstructs Float32 HU volumes — this
  is what gives the perfusion map real CT photon-noise statistics
* Final perfusion: voxel-wise Mullani–Gould on the recon-space V1 and V2

The BasisSim path produces images that look like real clinical CTP because
the noise is physical (Poisson photon stats + recon filter response), not
post-hoc Gaussian.

## Canonical SUBJECT005 demo (CFR ≈ 2.45, healthy)

Of 14 subjects in `UCLA_human_dataset`, **SUBJECT005** (PICC injection) is
the cleanest demo: REST AUC / STRESS AUC ratio = 2.23, falls within
clinical CFR 2.5–4.5 (`scan_all_aifs.py` ranks all 14).

End-to-end SUBJECT005 STRESS:
| stage | output | summary |
|---|---|---|
| `step0_prepare_aif.py` | `intermediate/aif_curve_subject005_stress.csv` | HU→mg/mL @ 35, drop trailing outlier, baseline-subtract |
| `step2_build_territory.jl` (hyperemic tree via env `STEP2_TREE_DIR`) | `myo_arrival.raw` (= ..._hyperemic) | per-voxel arrival 0.4 s median |
| `make_basissim_phantoms.jl` (V1 t=0 + V2 t=9, hyperemic arrival) | `basissim_phantoms_s005_stress/{v1_baseline,v2_peak}/phantom.raw` | UInt16 cross-product, 2.2 GB each |
| `run_cta_sim_param.jl` (kVp=100, mA=250) × 2 | `basissim_out_s005_stress/{v1_baseline,v2_peak}/recon_fbp_hu_f32.raw` | 512×512×250 Float32 HU, ~250 MB |
| `step5_perfusion_from_recon.py` (V2_t=9, --recon fbp) | `output/perfusion_map_subject005_stress.npy` | 4.38 mL/min/g mean, σ 2.51 |
| `smooth_downsample_viewer.py` | `output/perfusion_viewer_subject005_stress_full.html` | 5×5×5 mask-aware smoothing + opaque chamber mesh + cylinder tree |

Mirror run at REST (use at-rest tree + `aif_curve_..._rest.csv` + V2_t=16):
mean 1.79 mL/min/g → **CFR = 2.45×** (median 2.88×). Both states sit inside
the clinical healthy band (REST 0.5–1.5 / STRESS 2–5).

### 80 % proximal-LAD stenosis (intervention what-if)
`inject_lad_stenosis.py --stenosis-pct 80 --stenosis-len-mm 5` → narrows the
first 8 LAD trunk segments (cum length 5.95 mm) by ×0.2 in diameter.
Poiseuille R amplification 625× on those segments. Re-runs step2 + V2
voxelize + V2 BasisSim + step5:

| Territory | Stress (no stenosis) | + 80% LAD stenosis | ratio |
|---|---|---|---|
| **LAD** | 4.37 mL/min/g | **1.97** | **0.45×** ← critical defect |
| LCX | 4.88 | 4.80 | 0.98× |
| RCA | 3.90 | 3.79 | 0.97× |

The defect is isolated to the LAD territory by physics: stenosed proximal
Poiseuille R → 5.5× longer arrival in LAD myocardium → lower ΔHU at the
sampled V2 time → lower Mullani–Gould flow. LCX/RCA are mathematically
untouched.

## Scripts directory

### Canonical (this generation — real AIF + BasisSim)
| script | role |
|---|---|
| `step0_prepare_aif.py` | `.mat` (UCLA layout: `aif_vec`, `time_vec`) → `intermediate/aif_curve_subjectXXX_*.csv` with HU→mg/mL conversion (default 35 for 100 kVp), baseline subtraction, trailing-outlier removal |
| `step0b_extrapolate_aif.py` | optional gamma-variate fit + wash-out extrapolation (used to debug AUC truncation issues — see "known limits" below) |
| `step2_build_territory.jl` | builds `intermediate/myo_arrival.raw` (Float32, per-voxel arrival in s) + `myo_tree_id.raw` (UInt8 0/1/2/3 = none/LAD/LCX/RCA) + `myo_dist.raw` (cm). Reads `$STEP2_TREE_DIR` env var (default `../VascularTreeSim.jl/output`); uses the FlowContrastSim hemo with `CAP_BED_R_PER_100G = 0.0` (autoregulation surrogate in the new pipeline = drop override + use real per-voxel arrival) |
| `make_basissim_phantoms.jl` | builds V1 (t=0) + V2 (peak AIF time) cross-product UInt16 phantoms ready for BasisSim. AIF voxels (aorta lumen + label-19 LV + label-21 LA + label-26 trunk) get pure-blood (bin_b=100, bin_i = AIF(t)/iodine_max × 100); myocardium voxels get ECV=0.25 (bin_b=25, bin_i = AIF(t-arrival)/disp/iodine_max × 100). Use `--use-myo-arrival <path>` for per-voxel mode |
| `step5_perfusion_from_recon.py` | loads BasisSim Float32 HU recons (V1+V2), maps the 1600×1400×500 myo mask through reverse(y,z)+downsample-2 → recon grid (512×512×250), applies Mullani–Gould `flow = 60/AUC × (V2_mass - V1_mass)`. Writes `perfusion_map_*.npy` + `flow_map_*.npy` + `myo_mask_*.npy` + `summary_*.txt` |
| `make_basissim_phantoms.jl` etc. | already listed above |
| `scan_all_aifs.py` | scans all `SUBJECT*` dirs in UCLA_human_dataset, ranks REST/STRESS AUC ratios for CFR-candidate selection |
| `inject_lad_stenosis.py` | walks the LAD main trunk (largest-D child at each fork) for the first N mm of cumulative length, multiplies their `diameter_um` by `(1 - pct/100)`. Writes a new tree dir with narrowed LAD + verbatim LCX/RCA |

### Viewer assembly
| script | role |
|---|---|
| `build_chamber_mesh.py` | XCAT chambers (labels 19-22) + aorta lumen mask → marching cubes → `intermediate/chamber_mesh_recon.npz` (verts + faces in recon mm). Watertight, no decimation; binary_closing(2) + gaussian σ=1 to suppress pinholes |
| `extract_tree_overlay.py` | streams a tree CSV, keeps segments with `diameter_um ≥ MIN_DIAM_UM` (default 500), projects endpoints to recon mm coords (`NRB cm → phantom voxel → reverse(y,z) → downsample 2 → recon idx → mm`), writes flat NaN-separated arrays + per-segment `starts_mm`, `ends_mm`, `diameter_um`, `tree_id` |
| `smooth_downsample_viewer.py` | **final viewer**. 5×5×5 mask-aware mean smoothing of the perfusion map on the native 232k myo voxels (no downsample), random sub-sample 80k for display, opaque chamber mesh underneath, 3D cylinder tree (true per-segment diameter, 6-side polygons, 3 colors LAD/LCX/RCA), toggle buttons for each tree |
| `build_basissim_perfusion_viewer.py` | simpler viewer (no smoothing, no chamber, no tree) — useful for sanity-checks before the full assembly |
| `build_perfusion_viewer.py` | legacy viewer for the path-A (uniform-arrival) perfusion map |

## Intermediate files (per phantom — keep, rebuild as needed)

```
intermediate/
  phantom_baseline_HU.raw         Int16 1600×1400×500, no contrast (from step1)
  aorta_lumen_mask.raw            UInt8 1600×1400×500, 1 = aorta lumen (from step1)
  metadata.toml                   step1 outputs incl. legacy synthetic gamma
  myo_arrival.raw                 Float32 1600×1400×500, current per-voxel arrival (s)
  myo_arrival_hyperemic.raw       … snapshot for hyperemic tree (max-dilated)
  myo_arrival_atrest.raw          … snapshot for at-rest tree (scale_to_rest applied)
  myo_arrival_stenosis_lad80.raw  … snapshot post-stenosis (hyperemic + LAD narrowed)
  myo_tree_id.raw                 UInt8, 0/1/2/3 = none/LAD/LCX/RCA territory
  myo_dist.raw                    Float32, distance (cm) to nearest terminal midpoint
  chamber_mesh_recon.npz          verts/faces in recon mm — opaque blob for viewer
  tree_overlay_hyperemic.npz      ≥500 μm segments for STRESS viewer (max-dilated)
  tree_overlay_atrest.npz         ≥500 μm segments for REST viewer (at-rest)
  tree_overlay_hyperemic_stenosis.npz  ≥500 μm segments post-LAD-stenosis
  aif_curve_subjectXXX_{rest,stress}.csv   per-patient AIF curve in mg/mL
```

Each `myo_arrival_*` file is the same Float32 binary so they can be passed
interchangeably via `make_basissim_phantoms.jl --use-myo-arrival <path>`.

## BasisSim integration

The BasisSim runner lives in `../phantom_ct_input/run_basis_sim/`. Two entry
points:

* `run_cta_sim.jl` — original, hardcoded paths
* **`run_cta_sim_param.jl`** — CLI args `PHANTOM_DIR OUTPUT_DIR [--kvp 100] [--mA 250] [--no-dicom]`; **writes the FBP recon + `recon_meta.toml` immediately after FBP completes** (the HIR workspace setup has a Siddon bounds-error bug on this machine that crashes Julia; FBP is preserved, HIR is best-effort)

The parametric script supports kVp=100 (matches SUBJECT005's scan) and
re-uses BasisSim's existing scanner + protocol model.

## Known limits

1. **STRESS bolus-tracking AIF truncation**. The bolus tracker stops at peak
   (= scan trigger). For most patients we only have wash-in, no wash-out.
   `step0b_extrapolate_aif.py` does gamma fit to extrapolate wash-out but
   the fit is unconstrained at the rising edge — easily over-extrapolates.
   This bias inflates STRESS perfusion (lower AUC than true) → inflated
   CFR. Best mitigation is to pick patients whose AIF tracking captured
   wash-out (`scan_all_aifs.py`).

2. **HIR recon crash in BasisSim**. `create_hir_recon_workspace` triggers a
   `siddon_trace_ray` bounds error during weight pre-compute on this
   machine. Cause unknown (GPU mem? indexing in BasisSim?). FBP is the
   canonical recon used downstream; HIR is best-effort.

3. **Per-voxel arrival vs autoregulation**. With `CAP_BED_R_PER_100G = 0` in
   step2 the at-rest-tree arrival map is too short (median 1.24 s) and the
   hyperemic-tree arrival is even shorter (0.43 s). Real healthy
   autoregulation enforces a much more uniform arrival ~1–2 s in both
   states. We deliberately let the per-voxel anatomy drive the variation
   because that's what the BasisSim noise + LAD stenosis demo needs.
   The original `ARRIVAL_OVERRIDE_S = 1.5` in legacy `step3` is the
   uniform-arrival counterpart.

4. **Coord transform LV apex offset**. The tree was grown in NRB cm coords;
   the chamber mesh was built in recon mm via reverse(y,z) + downsample 2.
   Both pass through the same transforms (`extract_tree_overlay.py`
   `nrb_to_recon_mm`) but at 0.2 mm voxel resolution the tree can appear
   to "graze" the chamber wall by 1–2 voxels in the viewer. Anatomical;
   not a transform bug.

5. **`run_basissim_*.sh` clean-up**. The BasisSim Julia process hangs on
   HIR cleanup after FBP saves. The shell wrappers `pkill -f run_cta_sim_param`
   the process once FBP is on disk. The Python harness reports "exit
   code 144" — that's the bash pipe to `tee` exiting via the kill; the
   FBP output is intact.
